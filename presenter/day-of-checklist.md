# Day-Of Checklist — AVD on Azure Local

Complete 30 minutes before session start.

## Environment Verification

- [ ] Azure portal loads and all blades are accessible
- [ ] Session hosts show "Available" status in the host pool
- [ ] FSLogix profile share is accessible and has VHDx files
- [ ] GPU session host responds to `nvidia-smi` (if demoing GPU live)
- [ ] AVD Insights workbook loads with data
- [ ] Log Analytics queries return results
- [ ] Cost Management blade accessible

## Browser Setup

- [ ] Clear all browser tabs
- [ ] Open bookmarked tabs for each demo starting point:
  - Azure Compute Gallery
  - AVD Host Pools
  - RDP session to session host (or Bastion ready)
  - PowerShell session to cluster node
  - AVD Insights
  - Log Analytics workspace
  - Cost Management
- [ ] Set browser zoom to 125–150% for audience visibility
- [ ] Disable browser notifications
- [ ] Disable OS notifications (Focus Assist / Do Not Disturb)

## Demo Machine

- [ ] Test projector / screen share with portal and RDP session open
- [ ] Verify font size is readable from the back of the room
- [ ] Close unnecessary applications
- [ ] Mute phone and messaging apps

## Fallback Materials

- [ ] Backup PowerPoint with screenshots is accessible offline
- [ ] Fallback recordings are playable
- [ ] `assets/screenshots/` folder is current with latest captures
